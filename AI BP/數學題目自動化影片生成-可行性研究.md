## 1. 研究概要

### 目標

建立一條自動化管線，能夠：
1. 輸入一道數學題目（文字或圖片）
2. AI 自動生成講解腳本（中文）
3. AI 自動生成 3Blue1Brown 風格動畫程式碼
4. 自動渲染為影片 + 合成中文語音旁白
5. 輸出完整的教學影片

### 結論預覽

**可行性：高度可行。** 截至 2026 年 3 月，所有必要的技術元件均已成熟或接近成熟：
- Manim 動畫引擎已穩定且功能豐富
- LLM 已能可靠生成 Manim 程式碼（多個開源方案驗證）
- 中文 TTS 已達自然語音水準（CosyVoice 3.0、FishAudio S1）
- 多個學術研究與開源專案已證明端到端管線的可行性

## 3. 核心引擎：Manim 動畫系統

### 兩個版本

| 版本 | 維護者 | 適用場景 |
|------|-------|---------|
| **3b1b/manim** | Grant Sanderson 本人 | 個人創作用，API 較不穩定 |
| **ManimCommunity/manim** | 開源社群 | 推薦使用，穩定、文件完整、社群活躍 |

### Manim 核心能力

- **數學物件：** 圓、向量、函數圖、LaTeX 方程式、座標軸、3D 場景
- **動畫效果：** 移動、變形、縮放、淡入淡出、旋轉、繪製（Write/Create）
- **程式化控制：** 用 Python 腳本精確控制每一幀的動畫行為
- **輸出格式：** MP4 影片（透過 FFmpeg）
- **LaTeX 整合：** 原生支援數學公式渲染

### 為什麼選 Manim 而非其他方案

| 方案 | 優點 | 缺點 |
|------|------|------|
| **Manim** | 專為數學設計、精確控制、LaTeX 原生支援、LLM 生成生態最成熟 | 學習曲線陡、渲染速度較慢 |
| **Remotion** | React 生態、Web 友好、適合批量生成 | 非數學專用、需自建數學元件 |
| **Motion Canvas** | 類似 Manim 的 TypeScript 版 | 生態較小、LLM 訓練資料少 |
| **D3.js + 影片** | Web 原生、互動性強 | 非影片導向、需大量客製化 |

**結論：Manim Community Edition 是目前最佳選擇**，因為 LLM 生成 Manim 程式碼的生態最為成熟（多個學術論文和開源專案驗證）。

## 4. LLM 驅動的自動化管線

以下是目前已知的 6 個主要開源/學術方案：

### 4.1 Manimator（學術級方案）

> 論文：[Manimator: Transforming Research Papers into Visual Explanations](https://arxiv.org/abs/2507.14306)

**三階段管線：**

```
階段 1：場景描述生成          階段 2：程式碼生成           階段 3：渲染
┌──────────────────┐    ┌──────────────────┐    ┌──────────────┐
│ 輸入：            │    │ 結構化描述 →      │    │ Python 程式碼 │
│ • 自然語言提示    │ →  │ 程式碼 LLM 轉換   │ →  │ → Manim 渲染 │
│ • PDF 論文       │    │ (DeepSeek-V3)    │    │ → MP4 輸出   │
│                  │    │                  │    │              │
│ LLM:            │    │ 輸出：            │    │              │
│ • gemini-2.0-flash│   │ • 可執行 Python   │    │              │
│ • llama-3.3-70b  │    │ • Manim 場景類    │    │              │
└──────────────────┘    └──────────────────┘    └──────────────┘
```

**場景描述格式（Markdown）：**
- 主題辨識（Topic identification）
- 關鍵數學公式（LaTeX 格式）
- 視覺元素規格（Visual elements）
- 風格指南（Style guidelines）

**評測成績：**
- TheoremExplainBench 總分：**0.845**（超越其他模型）
- 元素佈局分數：**0.853**
- 失敗時自動重試（最多 8 次重新提示）

### 4.2 Code2Video（三代理架構）

> 論文：[Code2Video: A Code-centric Paradigm for Educational Video Generation](https://arxiv.org/abs/2510.01174)
> GitHub：[showlab/Code2Video](https://github.com/showlab/Code2Video)

**這是目前最先進的學術方案。**

**三代理架構（Tri-Agent）：**

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Planner    │ →  │    Coder     │ →  │    Critic    │
│   規劃代理    │    │   編碼代理    │    │   評審代理    │
├──────────────┤    ├──────────────┤    ├──────────────┤
│ • 序列化概念  │    │ • 結構指令 →  │    │ • 多模態回饋  │
│ • 安排範例    │    │   Manim 程式碼│    │ • 視覺錨點    │
│ • 規劃回顧    │    │ • 精確佈局    │    │ • 空間優化    │
│ • 教學流程    │    │ • 時序控制    │    │ • 學習目標    │
│              │    │ • 模組化程式碼 │    │   對齊檢查    │
└──────────────┘    └──────────────┘    └──────────────┘
```

**評測成績（MMMC Benchmark）：**
- 117 個長影片、13 個學科領域、456 個評估單元
- 比基礎 LLM 提升 **+40% TeachQuiz** 分數
- 比基礎 LLM 提升 **+50% 美觀度**
- 生成速度提升 **9.7 倍**
- 美觀度與學習效果相關係數：**r = 0.971**

**核心優勢：** 可擴展（模組化整合新視覺化）、可解釋（每個決策可審計）、可控制（程式化精確控制時序與佈局）。

### 4.3 Math-To-Manim（多代理管線）

> GitHub：[HarleyCoops/Math-To-Manim](https://github.com/HarleyCoops/Math-To-Manim)

**六代理管線：**

| 代理 | 角色 | 輸出 |
|------|------|------|
| **PrerequisiteExplorer** | 探索先備知識，建立知識樹 | 概念依賴圖 |
| **MathematicalEnricher** | 豐富數學內容，加入 LaTeX 方程式 | 數學規格文件 |
| **VisualDesigner** | 設計視覺元素與動畫規格 | 視覺設計文件 |
| **NarrativeComposer** | 撰寫 2000+ 字講解敘事 | 完整教學腳本 |
| **CodeGenerator** | 生成可執行 Manim 程式碼 | Python 程式碼 |
| **VideoReviewer** | 審查並優化最終影片 | 品質報告 |

**支援的 LLM 後端：**
- DeepSeek R1-Zero（主要）
- Google Gemini
- Grok3
- Claude（透過 Anthropic Agent SDK）
- Kimi K2.5（六代理 Swarm 協調）

**特色功能：**
- **逆向知識樹演算法：** 自動發現先備概念，從基礎到進階建構教學流程
- **Claude Code 整合：** 可作為 Claude Code 的 Skill 直接使用
- **55+ 驗證範例：** 已有 55 個以上可運作的動畫範例

### 4.4 Generative Manim（最易上手）

> GitHub：[marcelo-earth/generative-manim](https://github.com/marcelo-earth/generative-manim)
> 線上版：[generative-manim.vercel.app](https://generative-manim.vercel.app/)

**三組件架構：**

```
┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│ Prompt Generator │ →  │  Generate Video  │ →  │  Render Engine   │
│ 文字 → 程式碼     │    │  程式碼 → 影片    │    │  Manim 渲染      │
│                  │    │                  │    │                  │
│ 支援模型：        │    │                  │    │                  │
│ • GPT-4o        │    │                  │    │                  │
│ • Claude Sonnet  │    │                  │    │                  │
└──────────────────┘    └──────────────────┘    └──────────────────┘
```

**優勢：**
- 有 Vercel 部署的線上版，可直接在瀏覽器使用
- 不需要 Python/Manim 知識
- 自然語言輸入即可生成動畫
- 開源可自託管

### 4.5 manim-generator（反饋迴圈）

> GitHub：[makefinks/manim-generator](https://github.com/makefinks/manim-generator)

**核心設計：Code-Writer + Code-Reviewer 反饋迴圈**

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 用戶輸入  │ →  │Code-Writer│ →  │  渲染嘗試  │
└──────────┘     │  (LLM)   │     └────┬─────┘
                 └────▲─────┘          │
                      │            成功？
                      │           ┌────┴────┐
                 ┌────┴─────┐     │         │
                 │Code-Reviewer│  │  是 → 輸出 │
                 │  (LLM)   │←──│  否 → 錯誤 │
                 │+ 錯誤日誌  │   │    紀錄    │
                 └──────────┘    └─────────┘
```

- 編譯失敗時，將錯誤訊息回傳 LLM 重新生成
- 最多允許 8 次重試
- 利用執行日誌持續改進程式碼品質

### 4.6 Manim MCP Server（Claude 整合）

> GitHub：[abhiemj/manim-mcp-server](https://github.com/abhiemj/manim-mcp-server)

**與 Claude Desktop / Claude Code 直接整合：**

- 透過 MCP（Model Context Protocol）協議連接 Claude 與 Manim
- 使用者在 Claude 聊天中描述動畫需求
- Claude 自動生成 Manim 程式碼，透過 MCP Server 渲染
- 直接在聊天中返回 1080p MP4 影片
- 支援低/中/高品質渲染選項
- 附帶常用動畫模板

### 7.3 資料流設計

```python
# 虛擬碼：端到端管線

def generate_math_video(problem: str) -> str:
    """
    輸入：數學題目（文字或圖片路徑）
    輸出：最終影片檔案路徑
    """

    # 階段 1：解題與概念分析
    solution = llm_solve(problem)  # LLM 解題
    concepts = extract_prerequisites(solution)  # 知識樹

    # 階段 2：教學設計
    scene_plan = plan_scenes(solution, concepts)  # 場景規劃
    # 輸出格式：[{scene_id, concept, visuals, duration}, ...]

    # 階段 3：生成中文講解腳本
    script = generate_chinese_script(scene_plan)
    # 輸出格式：[{scene_id, text_zh, timing}, ...]

    # 階段 4：並行生成
    # 4a: Manim 程式碼生成 + 渲染
    manim_code = generate_manim_code(scene_plan)
    animation_mp4 = render_manim(manim_code)  # Manim → MP4

    # 4b: 中文語音合成
    audio_segments = []
    for segment in script:
        audio = tts_chinese(segment.text_zh)  # TTS → WAV
        audio_segments.append(audio)

    # 階段 5：合成
    final_video = merge_audio_video(
        video=animation_mp4,
        audio=audio_segments,
        subtitles=script  # 可選字幕
    )

    # 階段 6：品質檢查
    quality_report = critic_review(final_video, solution)
    if not quality_report.passed:
        return regenerate_with_feedback(quality_report)

    return final_video
```

## 8. 關鍵挑戰與解決方案

### 挑戰 1：LLM 生成的 Manim 程式碼經常無法編譯

| 問題 | 解決方案 |
|------|---------|
| API 版本不符 | 在 prompt 中明確指定 Manim Community Edition 版本 |
| 物件引用錯誤 | 使用 Code-Writer/Code-Reviewer 反饋迴圈 |
| 渲染失敗 | 錯誤訊息回傳 LLM 重新生成（最多 8 次重試） |
| 複雜場景失敗率高 | 拆分為多個簡單場景，逐一生成再拼接 |

### 挑戰 2：動畫與語音時間同步

| 問題 | 解決方案 |
|------|---------|
| 語音長度與動畫不一致 | 先生成語音 → 測量各段時長 → 據此調整 Manim 動畫時間 |
| 中文語速不均 | TTS 引擎調整語速參數（rate/speed） |
| 場景切換不自然 | 加入過渡動畫（FadeIn/FadeOut）填補時間差 |

### 挑戰 3：數學正確性

| 問題 | 解決方案 |
|------|---------|
| LLM 解題錯誤 | 使用最強數學推理模型（GPT-5.2 AIME 99%、Claude Opus） |
| 公式渲染錯誤 | LaTeX 語法自動校驗 |
| 視覺化不精確 | Critic 代理比對解題步驟與動畫內容 |

### 挑戰 4：中文數學術語 TTS

| 問題 | 解決方案 |
|------|---------|
| 數學符號朗讀 | 腳本中將符號轉為中文描述（如 "∫" → "積分"） |
| 專業術語發音 | 使用教育領域微調的 TTS 模型或自訂發音詞典 |
| 中英混合 | CosyVoice/FishAudio 支援跨語言混合 |

### 挑戰 5：批量生成的一致性

| 問題 | 解決方案 |
|------|---------|
| 風格不一致 | 建立標準化的場景模板與色彩方案 |
| 品質波動 | 固定 Manim 動畫模板 + 嚴格品質檢查 |
| 講解深淺不一 | 在 prompt 中明確指定目標學生程度 |
