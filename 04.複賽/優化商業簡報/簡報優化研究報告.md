# Mathify 複賽簡報優化研究報告

> 創業綻放-創業大聯盟競賽 | 商業簡報最佳實踐研究

---

## 一、複賽官方規格

- **簡報頁數**：15 頁以內
- **複賽形式**：實體現場簡報與 Q&A
- **時間分配**：12 分鐘簡報 + 5 分鐘評審提問 + 8 分鐘團隊回應（共 25 分鐘）
- **評選觀點**：以創投觀點為核心進行綜合評估
- **繳交期限**：115 年 3 月 31 日前繳交優化的商業簡報及商業計畫書

### 五大評分面向（各 20%）

1. 解決方案
2. 市場分析
3. 商業模式
4. 團隊潛力
5. 財務規劃與管理

## 二、競賽簡報 vs VC 募資簡報的關鍵差異

| 面向 | 創業競賽簡報 | VC 募資簡報 |
|------|-----------|-----------|
| **受眾** | 評審委員（多元背景、創投觀點） | 特定投資人（財務回報導向） |
| **目標** | 獲得獎金/支持金、證明團隊潛力 | 獲得股權投資、建立長期關係 |
| **時間** | 12 分鐘簡報 + 13 分鐘 Q&A | 通常 30-60 分鐘深度討論 |
| **財務深度** | 高層級但須展現合理性 | 要求詳細的單位經濟學與回報模型 |
| **重點** | 創新性、社會影響力、團隊執行力 | 市場規模、獲利路徑、退出策略 |
| **Traction** | 軟指標也可以（問卷、訪談、合作意向） | 必須有硬指標（MRR、用戶增長） |
| **結尾** | 「我們需要 NT$300 萬來實現...」明確 CTA | 「我們正在募資 X 輪」 |
| **整體調性** | 熱情 + 數據 + 使命感 | 自信、數據驅動 |

**關鍵洞察**：競賽評審看的是「這個團隊是否值得支持」，而非「我能賺多少倍回報」。

## 三、現有簡報弱點分析

### 3.1 評分面向覆蓋失衡

| 評分面向 (各20%) | 現有對應投影片 | 佔比 | 問題 |
|-----------------|-------------|------|------|
| **解決方案** | #2 痛點, #3 產品, #4 創新性, #5 競爭力 | 4/15 = 27% | 比例偏高，但缺乏數據佐證與用戶驗證 |
| **市場分析** | #6 行業現況, #7 市場規模, #8 拓展性 | 3/15 = 20% | 市場規模只有 2 個數字，缺 TAM/SAM/SOM |
| **商業模式** | #9 商業模式, #10 可行性 | 2/15 = 13% | **嚴重不足**，缺 GTM 策略與單位經濟學 |
| **團隊潛力** | #11 團隊, #12 創業者經驗 | 2/15 = 13% | 需展示互補性與執行力，而非只列經歷 |
| **財務規劃** | #13 資金規劃, #14 KPI | 2/15 = 13% | **嚴重不足**，缺財務預測、損益表、現金流 |
| 其他 | #1 封面, #15 願景 | 2/15 = 13% | 封面與結尾 |

**核心問題**：解決方案佔比過高（27%），而商業模式、團隊潛力、財務規劃各只佔 13%，與五大面向各 20% 的權重嚴重失衡。

### 3.2 逐頁弱點

| 投影片 | 問題 | 改善方向 |
|--------|------|---------|
| #2 學習痛點 | 三大斷層缺乏數據佐證 | 加入會考成績分布、城鄉差距數據 |
| #3 產品服務 | 太抽象，無產品畫面 | 加入產品原型截圖/Demo |
| #4 創新性 | 只與 ChatGPT 比較 | 改為 2x2 競爭定位矩陣 |
| #7 市場規模 | **只有 2 個數字** | 改為完整 TAM/SAM/SOM 漏斗圖 |
| #9 商業模式 | 太簡略 | 加入定價層級、單位經濟學、GTM |
| #10 可行性 | 定位模糊 | 拆為「GTM 策略」+ 刪除或併入其他頁 |
| #11+#12 團隊 | 分散兩頁 | 合併為一頁，強調互補性 |
| #13 資金規劃 | 只有分配表 | 加入三年財務預測、盈虧平衡 |
| #14 KPI | 與財務脫節 | 加入 SaaS 指標（MRR、ARR、流失率） |
| #15 願景 | 只說「謝謝」 | 改為強力 Call to Action |

### 3.3 缺失的關鍵元素

1. **Traction / 早期驗證數據** — 最關鍵的缺失
2. **產品 Demo / 原型畫面** — 必須加入
3. **風險分析與應對策略** — 展現團隊成熟度
4. **GTM（市場進入策略）** — 商業模式的關鍵拼圖
5. **完整的財務模型** — 三年損益預測、盈虧平衡

## 四、台灣創業競賽簡報六大致命傷

來源：BFA 簡報

1. **缺乏聽眾分析** — 沒有預先調查評審背景
2. **內容缺乏事實佐證** — 使用個人意見而非數據和事實
3. **簡報缺乏市場驗證** — 沒有 Traction（使用數據、測試結果）
4. **設計濫用色彩** — 應只用品牌色搭配黑白灰
5. **結尾缺乏行動呼籲** — 只說「謝謝」而非強烈 CTA
6. **太早參賽** — 在沒有實際成績的情況下參賽

## 五、建議的優化投影片結構（15 頁）

### 設計原則

1. **五大面向均衡覆蓋**：每個評分面向至少 2-3 頁
2. **一頁一觀點**：每張投影片只傳達一個核心訊息
3. **數據為王**：每個論點都要有數據或事實佐證
4. **敘事弧線**：問題 → 解決方案 → 市場 → 商業模式 → 團隊 → 財務 → CTA

### 15 頁結構

```
#1  封面                    [品牌]    15秒
#2  問題/學習痛點            [解決方案] 1.5分鐘
#3  解決方案/產品展示        [解決方案] 2分鐘
#4  創新性與競爭優勢         [解決方案] 1.5分鐘
#5  行業現況與市場趨勢       [市場分析] 1分鐘
#6  市場規模 TAM/SAM/SOM    [市場分析] 1分鐘
#7  市場拓展與國際化路徑     [市場分析] 45秒
#8  商業模式與定價           [商業模式] 1.5分鐘
#9  市場進入策略 GTM         [商業模式] 1分鐘
#10 早期驗證 / Traction     [解決方案+商業模式] 1分鐘
#11 團隊                    [團隊潛力] 1分鐘
#12 財務預測                [財務規劃] 1分鐘
#13 資金運用與里程碑         [財務規劃] 1分鐘
#14 風險分析與應對           [團隊潛力+商業模式] 45秒
#15 願景與行動呼籲 CTA      [全部]    30秒
                                      合計 ~14分鐘（含緩衝）
```

### 優化後覆蓋分析

| 評分面向 (各20%) | 對應投影片 | 佔比 | 改善 |
|-----------------|-----------|------|------|
| **解決方案** | #2, #3, #4, #10 | 4/15 = 27% | 加入驗證數據與 Demo |
| **市場分析** | #5, #6, #7 | 3/15 = 20% | 完整 TAM/SAM/SOM + 拓展路徑 |
| **商業模式** | #8, #9, #10, #14 | 3-4/15 = 20-27% | 加入 GTM + 單位經濟學 |
| **團隊潛力** | #11, #14 | 2/15 = 13% | 強調互補性 + 風險展現成熟度 |
| **財務規劃** | #12, #13 | 2/15 = 13% | 完整三年預測 + 資金分配 |

## 七、設計與視覺化技巧

### 數據視覺化

- **用圖表取代子彈列表**：營收用折線圖、市場用漏斗圖、資金用餅圖、競爭用 2x2 矩陣
- **大數字突出法**：關鍵統計用超大字體（48pt+），搭配簡短解釋
- **顏色一致性**：品牌色（#1B3A5C / #2C5E8A）+ 黑白灰，重點用對比色

### 設計原則

- 每頁不超過 6 行文字
- 善用留白，不要塞滿
- 圖表 > 表格 > 文字
- 品牌色一致性（深藍色系）

## 八、Q&A 準備建議

### 評審可能問的問題

1. AI 模型的數學準確率如何保證？與 ChatGPT 相比具體優勢在哪？
2. 為什麼選擇 Freemium 而不是直接收費？轉換率假設的依據？
3. 如何與均一教育平台、康軒數位產品區隔？
4. 少子化趨勢下，市場是否會萎縮？
5. NT$300 萬夠用嗎？如果燒完了怎麼辦？
6. 兩人團隊如何同時做開發、行銷、營運？
7. B2B2C 學校端合作的具體策略？
8. 有做過任何用戶測試嗎？結果如何？
9. 如果大型平台（如 Google、均一）也做 AI 數學，你們怎麼應對？
10. 三年後如果沒有達到預期目標，你們的 Plan B 是什麼？

### 建議準備備用投影片（Appendix）

- 詳細財務報表（損益表、現金流量表）
- 技術架構圖
- 競品詳細對比表
- 產品路線圖詳細版
- 用戶訪談/問卷調查原始數據

## 九、最高優先行動清單

### Must-Do（必做）

1. ✅ 重建市場規模頁 → 完整 TAM/SAM/SOM 漏斗圖
2. ✅ 加入三年財務預測 → 損益表 + 盈虧平衡 + SaaS 指標
3. ✅ 加入 Traction / 驗證數據 → 即使是軟指標也好
4. ✅ 改造結尾為 CTA → 明確說明需求與承諾

### Should-Do（應做）

5. 加入產品原型 Demo 畫面
6. 加入 GTM 策略頁
7. 合併團隊頁，強調互補性
8. 加入風險分析
9. 競爭定位改為 2x2 矩陣圖

### Nice-to-Have（加分）

10. 預先調查評審背景
11. 準備備用投影片（Q&A 用）
12. 練習簡報至少 10 次
