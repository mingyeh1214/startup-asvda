# 三、商業模式

> **假設基礎聲明**：Mathify Labs 目前處於技術原型階段（已有 AI 對話 + 前端介面，視覺化引擎開發中），尚無營運數據。以下所有數字均為推估，每項假設均標註推導邏輯與參考依據。

## (一) 商業模式

### 1. 收費機制與定價邏輯

**Freemium SaaS 訂閱制**——免費方案拉新、付費方案變現。


| 方案       | 月費             | 功能                           | 設計邏輯          |
| -------- | -------------- | ---------------------------- | ------------- |
| **免費方案** | NT$0           | 每日 3 次 AI 對話 + 基礎題庫          | 降低嘗試門檻，創造口碑傳播 |
| **付費方案** | NT$600/月       | 無限 AI 對話 + 完整題庫 + 模擬考 + 學習報告 | 單一價格降低決策成本    |
| **年繳方案** | NT$500/月（83 折） | 同付費方案                        | 提高留存、穩定現金流    |


**為什麼訂 NT$600？**


| 對比對象         | 月費              | Mathify 定位              |
| ------------ | --------------- | ----------------------- |
| 傳統補習班        | NT$3,000–8,000  | **不到十分之一**，取代性價比極高      |
| 一對一家教        | NT$8,000–20,000 | 完全不同量級                  |
| ChatGPT Plus | ~NT$640         | 價格相當，但 Mathify 專為台灣課綱設計 |
| Khanmigo     | ~NT$130         | 英文產品，無法對應 108 課綱        |


**結論**：NT$600 落在「學生零用錢可負擔、家長願意替代補習」的甜蜜區間。實際定價將在 MVP 上線後透過 A/B 測試驗證。

### 2. 單位經濟模型

以下從**定價 → 成本 → 利潤 → 客戶價值 → 獲客成本**逐層推導，每層都標註假設來源。

#### Step 1：付費用戶月均收入（ARPPU）

- 單一定價方案：**ARPPU = NT$600/月**
- 若未來 20% 用戶選擇年繳（83 折），有效 ARPPU 降至約 NT$580，此處保守仍以 NT$600 計算

> 備註：ARPU（含免費用戶）= ARPPU × 免費→付費轉換率。以 Year 1 目標 3.3% 轉換率計算，ARPU ≈ NT$20/月。Freemium 模式下 ARPU 低是正常特徵，Duolingo 的 ARPU 僅約 US$1.5/月。

#### Step 2：每用戶變動成本（COGS）


| 成本項目        | 月均/付費用戶     | 假設依據                                                                                       |
| ----------- | ----------- | ------------------------------------------------------------------------------------------ |
| AI 推理成本     | NT$80–120   | 以 Claude Sonnet API 定價估算：平均 150 次查詢/月 × ~600 tokens/次 × 輸入輸出加權單價。使用快取與模型路由可壓至 NT$0.5–0.8/次 |
| 雲端基礎設施      | NT$15–25    | AWS/GCP 基礎方案分攤至單用戶，含資料庫、CDN、儲存                                                             |
| **COGS 合計** | **~NT$120** | 取中位數，隨規模擴大有議價空間                                                                            |


- **單位貢獻利潤** = NT$600 − NT$120 = **NT$480/月**
- **貢獻利潤率** = 480 ÷ 600 = **80%**（SaaS 業界健康基準 > 70%）

> ⚠️ 此為推估值。實際 COGS 需待 MVP 上線後以真實使用數據校準。AI 推理成本受模型選擇、快取命中率、用戶查詢量影響甚大——這也是為什麼 M3 Alpha 測試的重要目的之一就是驗證 COGS 假設。

#### Step 3：客戶生命週期

教育市場不適用傳統 SaaS 的 1/月流失率 公式——學生會畢業。


| 留存假設      | 占比  | 平均留存  | 邏輯             |
| --------- | --- | ----- | -------------- |
| 短期（段考衝刺型） | 60% | 10 個月 | 考前訂閱、考後退訂，約一學年 |
| 中期（持續學習型） | 30% | 20 個月 | 跨學年使用，升年級續訂    |
| 長期（升學銜接型） | 10% | 30 個月 | 國中→高中持續使用      |


- **加權生命週期** = 0.6×10 + 0.3×20 + 0.1×30 = **15 個月**
- 考量寒暑假流失（學期間流失率約 15–20%），保守取 **14 個月**

> 假設依據：台灣補習班業界經驗，學生平均補習持續 1–2 學年。線上訂閱的退訂摩擦力更低，故取保守值。

#### Step 4：LTV 與 CAC


| 指標            | 數值           | 推導                  |
| ------------- | ------------ | ------------------- |
| **LTV**       | **NT$6,720** | 貢獻利潤 NT$480 × 14 個月 |
| **CAC**（加權平均） | **NT$500**   | 見下方渠道分析             |
| **LTV/CAC**   | **13.4:1**   | 遠高於 3:1 健康線         |
| **CAC 回收期**   | **1.0 個月**   | NT$500 ÷ NT$480     |


**CAC 分渠道假設**：


| 獲客渠道       | 單位 CAC    | 占比  | 邏輯                      |
| ---------- | --------- | --- | ----------------------- |
| 免費→付費（PLG） | ~NT$100   | 40% | 免費用戶服務成本分攤，產品自然轉換       |
| 口碑推薦       | ~NT$400   | 25% | 推薦獎勵機制（贈送付費天數）          |
| KOL/社群     | ~NT$1,000 | 20% | 數學教育 YouTuber 合作，CPA 模式 |
| 校園推廣       | ~NT$650   | 15% | 教育展 + 學校免費試用            |


加權 CAC = NT$438，取 **NT$500**（含測試與優化成本）。

> LTV/CAC 13.4:1 偏高的原因：65% 獲客來自 PLG 和口碑（近乎零成本）。這是 EdTech 特有的學生社交傳播效應——一個學生用了覺得好，會直接傳給同班同學。若未來需加大付費渠道（KOL、廣告），CAC 會顯著上升，但即使 CAC 翻倍至 NT$1,000，LTV/CAC 仍有 6.7:1。

#### 敏感度分析


| 情境    | 生命週期  | CAC      | LTV/CAC | 說明             |
| ----- | ----- | -------- | ------- | -------------- |
| 🟢 樂觀 | 18 個月 | NT$400   | 21.6:1  | 留存優於預期、PLG 效果佳 |
| 🟡 基準 | 14 個月 | NT$500   | 13.4:1  | 當前假設           |
| 🔴 悲觀 | 10 個月 | NT$1,000 | 4.8:1   | 留存差、需靠付費渠道獲客   |


即使悲觀情境，LTV/CAC 仍 > 3:1。**單位經濟的安全邊際足夠。**

### 3. 毛利率


| 類別           | 項目          | 月均成本          | 性質             |
| ------------ | ----------- | ------------- | -------------- |
| **COGS**（變動） | AI 推理 + 雲端  | ~NT$120/付費用戶  | 隨用戶數線性增長       |
| **OpEx**（固定） | 人事、行銷、內容、行政 | ~NT$16–40 萬/月 | 不隨用戶數變動（依團隊規模） |


**毛利率 = (NT$600 − NT$120) ÷ NT$600 = 80%**

關鍵特性：毛利率 80% 不隨規模變化。隨付費用戶成長，固定成本被攤薄 → 淨利率持續改善。這是 SaaS 模式的核心：**邊際服務成本極低，規模即利潤。**

### 4. 通路與品牌策略

**Product-Led Growth（產品驅動增長）**：讓產品本身成為最好的行銷。

核心飛輪：免費用戶體驗 → 口碑傳播 → 免費用戶成長 → 付費轉換 → 資源投入產品 → 體驗更好

**品牌定位**：不是冷冰冰的「AI 工具」，而是**「每個孩子身邊那位最有耐心的數學老師」**。

## (二) 三年財務預測

### 方案一：NT$300 萬

**假設鏈**：

- 用戶成長：Y1 500 → Y2 2,000 → Y3 5,000 付費用戶（年底）
- Y2/Y3 成長動力：Y1 驗證 PMF 後，以種子輪資金（NT$500–1,000 萬）加速
- ARPPU 固定 NT$600/月（單一定價）
- COGS 固定 NT$120/月/用戶（規模效應暫不假設）


| 項目         | Year 1       | Year 2       | Year 3         |
| ---------- | ------------ | ------------ | -------------- |
| 付費用戶（年底）   | 500          | 2,000        | 5,000          |
| 月均付費用戶     | ~150         | ~1,250       | ~3,500         |
| **年度營收**   | **NT$110 萬** | **NT$900 萬** | **NT$2,520 萬** |
| COGS（變動成本） | NT$13 萬      | NT$180 萬     | NT$504 萬       |
| OpEx（固定成本） | NT$192 萬     | NT$420 萬     | NT$840 萬       |
| 總成本        | NT$205 萬     | NT$600 萬     | NT$1,344 萬     |
| **淨利/虧損**  | **-NT$95 萬** | **NT$300 萬** | **NT$1,176 萬** |
